Revised coupon architecture

Coupons are reusable across unlimited orders by any customer, including repeated use by the same customer. Only stacking multiple coupons within one checkout is prohibited.There is no usage limit, reservation, redemption counter, customer tracking, or coupon-consumption lifecycle. No discount_redemptions table is needed.

Final behaviorNormalize codes with trim().toUpperCase().Permit exactly one couponCode string per checkout.Reject arrays, multiple code fields, or malformed combined codes.Reapplying the same code recalculates from original server product prices; it never applies a discount to an already-discounted value.Products configured for the code receive their assigned percentage.Unconfigured products remain full price.Shipping remains full price.Reject a coupon when the total rounded discount is zero.Preview is advisory. Checkout independently reloads all data and recalculates.Invalid, inactive, future, expired, or inapplicable coupons fail checkout; the server must not silently continue at full price.Coupon changes affect future Checkout Sessions. A Session already created keeps the exact prices the server sent to Stripe.Completed orders become the permanent coupon usage history.
